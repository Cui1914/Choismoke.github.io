# 烟灰缸本地数据层说明

这份文档只讲“真实数据保存第一版”现在做到了什么。

## 当前已经补上的内容

- 本地账号数据文件
- 本地论坛数据文件
- 统一读写工具
- 仓库层读取本地数据
- 发帖和评论会写入本地文件

## 当前本地数据文件

- `community-app/data/auth-db.json`
- `community-app/data/forum-db.json`

## 这代表什么

现在论坛不再只是“接口占位”。

至少在第一版里：

- 发帖已经有本地落盘位置
- 评论已经有本地落盘位置
- 账号信息已经有本地数据文件位置

## 当前这一层的意义

这不是最终上线方案，但它有两个价值：

### 1. 先让数据不再只是写死在页面里

也就是说，项目已经从“完全静态演示”往“真的会保存东西”迈出了一步。

### 2. 为后面接真实数据库做过渡

等后面换成真正的数据库时，不需要重写页面，只需要替换数据读取和保存的位置。

## 你现在只需要知道的结论

我已经开始把烟灰缸从“页面原型”推进到“会真的存数据的第一版”。
