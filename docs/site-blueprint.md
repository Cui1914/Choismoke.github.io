# 网站定位与板块说明文档

## 1. 网站定位

- 网站类型：个人网站
- 核心目标：展示计算机项目作品，并将访客引导到生活内容与社交账号关注
- 核心受众：
  - 自己（长期记录）
  - 朋友/同好（了解近况与兴趣）
  - 潜在合作方（评估能力与风格）

## 2. 用户路径

1. 访客进入首页，快速理解你是谁、做什么。
2. 访客点击作品集，查看项目能力与思考。
3. 访客继续浏览生活内容，感受到你的持续性与个人风格。
4. 访客通过社交入口关注你。

## 3. 信息架构

- 首页（`index.html`）
  - 顶部简介（价值主张）
  - 精选项目预览
  - 生活记录预览
  - 社交关注入口
- 作品页（`portfolio.html`）
  - 项目列表（可按标签筛选）
  - 每个项目展示：背景、目标、方案、成果、反思
  - 跳转项目详情页（`project-template.html` / `project-sample.html`）
- 生活页（`life.html`）
  - 周更记录（学习、运动、音乐、游戏、美食、旅游、思考、读书观影）
  - 可持续追加新周卡片
  - 周记模板区（可复制后直接填写）
- 关于页（`about.html`）
  - 个人简介
  - 兴趣图谱
  - 联系方式
- 全站配置（`config.js`）
  - 社交链接统一配置，避免多页面重复改动

## 4. 内容规范

### 4.1 作品内容模板

每个项目保持以下字段：

- 项目名
- 一句话定位
- 场景/问题
- 你的方案（核心技术思路）
- 结果（功能、性能或用户反馈）
- 复盘（你学到了什么）

### 4.2 生活周更模板

- 本周关键词（3-5个）
- 学习进展（1段）
- 运动记录（1段）
- 娱乐与文化（音乐/游戏/读书观影）
- 一次思考（短观点）
- 下周计划（3条）

## 5. 视觉与交互原则

- 视觉风格：极简、有留白、高级感
- 交互原则：交互服务信息浏览，不做干扰性动画
- 强调层级：标题、摘要、卡片动作反馈清晰

## 6. 运营节奏（每周）

1. 更新 `life.html` 新的一周记录。
2. 在首页“最新动态”替换为本周摘要。
3. 若有新项目，在 `portfolio.html` 增加项目卡片并补充标签。
4. 在 `config.js` 统一维护社交链接并检查可用性。

## 7. 后续可迭代方向（非当前必须）

- 增加文章详情页（把生活记录拆成独立文章）
- 增加项目详情页（从卡片跳转完整案例）
- 接入内容管理系统（便于长期维护）
- 增加中英双语版本
