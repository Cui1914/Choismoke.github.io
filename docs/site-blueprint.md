# 网站定位与板块说明文档

## 1. 网站定位

- 网站类型：个人网站
- 品牌名：Choismoke
- 核心目标：展示计算机项目作品，并将访客引导到生活内容、社交账号与后续论坛版块
- 核心受众：
  - 自己（长期记录）
  - 朋友与同好（了解近况与兴趣）
  - 潜在合作方（判断能力与风格）

## 2. 用户路径

1. 访客进入首页，快速理解网站主旨。
2. 访客先进入作品页，看到能力展示。
3. 访客继续浏览生活记录与关于页，建立对人的认知。
4. 访客进入论坛版块“烟灰缸”，从浏览转为互动。

## 3. 信息架构

- 首页：`index.html`
  - 顶部简介
  - 精选项目
  - 生活内容预览
  - 社交入口
- 作品页：`portfolio.html`
  - 项目列表
  - 标签筛选
  - 项目详情跳转
- 生活页：`life.html`
  - 周更记录
  - 周记模板
- 关于页：`about.html`
  - 个人简介
  - 兴趣图谱
  - 联系方式
- 论坛首页：`forum.html`
  - 置顶公告
  - 分类入口
  - 排序切换
  - 帖子流
  - 侧边信息模块
- 全站配置：`config.js`
  - 社交链接统一配置

## 4. 内容规范

### 4.1 作品模板

- 项目名
- 一句话定位
- 场景 / 问题
- 方案
- 结果
- 复盘

### 4.2 生活模板

- 本周关键词
- 学习进展
- 运动记录
- 娱乐与文化
- 一次思考
- 下周计划

### 4.3 论坛文档

- 论坛需求在 `docs/forum-spec.md`
- 论坛实现边界在 `docs/forum-implementation-notes.md`

## 5. 视觉与交互原则

- 主站与论坛保持统一视觉语言
- 视觉风格保持极简、高级感、克制交互
- 论坛允许更高信息密度，但不能破坏主站气质

## 6. 运营节奏

1. 每周更新 `life.html`。
2. 有新项目时更新 `portfolio.html` 与对应详情页。
3. 论坛进入动态开发前，先持续维护 `docs/forum-spec.md`。
4. 社交链接只在 `config.js` 中统一维护。

## 7. 后续方向

- 增加论坛列表页、帖子详情页、登录注册页、资料页前端模板
- 将论坛从静态前端骨架迁移到动态应用
- 最终实现统一账号体系与社区能力
