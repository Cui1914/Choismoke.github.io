# 烟灰缸论坛需求说明

## 1. 产品定位

- 名称：烟灰缸
- 所属关系：挂在 `Choismoke` 主站中的独立社区子系统
- 目标：让访客公开浏览、注册登录后发帖互动，形成持续社区内容
- 风格方向：延续主站的极简、高级感，信息密度适中

## 2. 访问与账号

- 游客：可公开浏览帖子、分类页、用户资料页
- 登录方式：邮箱注册登录
- 邮箱规则：校验邮箱格式，提供常见后缀提示，不强制邮箱验证
- 账号体系：与整站共用同一套账号

## 3. 用户能力

- 发主题帖
- 回复评论
- 点赞
- 收藏
- 分享链接
- 编辑自己的帖子和评论
- 删除自己的帖子和评论
- 上传图片
- 上传视频
- 插入代码块
- 使用标签
- 发布投票
- 私信
- 举报内容
- 关注他人

## 4. 内容结构

- 主分类：
  - 技术
  - 生活
  - 游戏
  - 随便聊
- 首页结构：
  - 固定置顶公告区
  - 分类入口
  - 排序切换：综合 / 最新 / 最热
  - 帖子流
  - 搜索框
  - 热门标签
  - 活跃用户
  - 最新公告
  - 今日热帖
  - 新人报道入口

## 5. 排序规则

- 综合：社区自然流，热度与新鲜度折中
- 最新：按发布时间倒序
- 最热：点赞、浏览、收藏加权

## 6. 帖子与评论规则

- 帖子标题：最长 17 字
- 帖子正文：最长 300 字
- 每帖图片：最多 10 张
- 每帖标签：最多 5 个
- 每帖投票选项：最多 7 个
- 评论长度：最长 50 字
- 评论支持：图片、代码块、楼中楼回复
- 编辑：帖子和评论都不限次编辑，但显示“已编辑”
- 删除：直接删除，不进回收站
- 标签来源：用户自由创建
- 图片和视频：都以文件形式上传到帖子中

## 7. 用户资料规则

- 用户名：2-10 位
- 用户名允许字符：中文、英文、数字、下划线
- 用户名修改次数：最多 10 次
- 头像：允许自定义上传
- 个人简介：最多 50 字
- 社交链接字段：
  - 邮箱
  - Bilibili
  - GitHub
- 资料页展示：
  - 用户名
  - 头像
  - 个人简介
  - 注册时间
  - 发帖数
  - 回复数
  - 获赞数
  - 最近发帖
  - 社交链接
  - 关注 / 粉丝

## 8. 社交与私信

- 关注关系：单向关注，互相关注视为好友
- 关注 / 粉丝列表：用户可选是否公开
- 私信规则：
  - 互关前允许陌生人发送 1 条私信
  - 单条私信最长 200 字
  - 私信允许图片
  - 支持拉黑用户

## 9. 举报与处罚

- 举报原因固定项：
  - 感到不适
  - 内容不实
- 举报后：内容先隐藏
- 用户累计被举报 5 次：自动限制
- 封禁类型：管理员审核后决定 7 天封禁或永久封禁
- 不启用预审核
- 不启用敏感词自动拦截
- 用户名不允许重复

## 10. 互动数据

- 帖子详情展示：
  - 浏览数
  - 点赞数
  - 评论数
  - 收藏数
  - 分享数
  - 发布时间
  - 编辑时间
  - 作者信息卡片
- 收藏：仅自己收藏夹可见
- 分享：复制帖子链接

## 11. 通知与后台

- 通知类型：
  - 回复你
  - 点赞你
  - 关注你
  - 举报处理结果
  - 系统公告
- 后台能力：
  - 用户管理（封禁、禁言）
  - 删帖删评论
  - 置顶 / 加精
  - 分类管理
  - 举报处理
  - 公告发布
  - 基础数据统计

## 12. 当前实现边界

- 当前仓库继续托管在 GitHub 上没有问题
- 但邮箱注册、发帖、评论、私信、通知、后台都需要服务端与数据库
- GitHub Pages 只能承载论坛前端静态壳和公开展示页，不能独立完成真实论坛业务
