Project Case

天气速览页

参考开源天气应用，强调“关键信息一眼可读”和流畅切换城市。

项目概览

  • 周期：1 周
  • 角色：独立开发
  • 类型：数据展示页面
  • 标签：天气数据 / 视觉层级 / 城市切换

背景与目标

背景：天气类页面常有信息过多、重点不清的问题。

目标：突出实时温度和未来趋势，降低阅读负担。

方案与实现

  • 模块 1：实时天气卡片（温度、体感、天气状态）
  • 模块 2：未来预报列表（日期、区间温度、图标）
  • 模块 3：城市切换和最近城市缓存

结果与复盘

结果：用户可在首屏快速判断当天出行准备。

复盘：后续可加入空气质量和降雨概率图表。
